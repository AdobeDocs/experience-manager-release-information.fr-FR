# aem [!DNL Forms] versions {#aem-forms-releases}

Adobe Experience Manager [!DNL Forms] est une application déployée sur AEM. Les formulaires AEM étendent les contenus Web et mobiles attrayants aux formulaires et documents d’entreprise, ce qui permet aux entreprises de procéder à des interactions numériques complexes tout en améliorant l’interaction des utilisateurs et en étendant la portée de l’entreprise. L&#39;AEM [!DNL Forms] prêt à l&#39;emploi inclus dans AEM Quick début a un ensemble limité de fonctionnalités. aem package de module complémentaire [!DNL Forms] fournit des fonctionnalités avancées telles que l&#39;intégration avec [!DNL Adobe Analytics], [!DNL Adobe Sign], les services de document, le moteur de workflow, et bien d&#39;autres.

>[!NOTE]
>
>AEM Forms publie les packages de modules complémentaires une semaine après la date de publication du Service Pack AEM et du Cumulative Fix Pack.

La liste suivante contient tous les packages de modules complémentaires [!DNL Forms] AEM publiés jusqu&#39;à aujourd&#39;hui, les versions AEM correspondantes (Conditions préalables), les liens de téléchargement des packages et d&#39;autres informations utiles :

* Les modules complémentaires AEM Forms répertoriés ci-dessus s’appliquent à la fois à AEM [!DNL Forms] sur OSGi et à AEM [!DNL Forms] sur JEE.
* Tous les packages de modules complémentaires [!DNL Forms] et les Quick Fixes AEM sont cumulatifs ; c&#39;est-à-dire qu&#39;un module complémentaire et un Quick Fix incluent tous les modules complémentaires et Quick Fixes précédents pour un composant pour la version correspondante. Par exemple, si vous installez Quick Fix 2.4.50, il inclut déjà AEM package de module complémentaire [!DNL Forms] 2.4.26.

   >[!NOTE]
   >
   >Les packages de modules complémentaires cumulés et les Quick Fixes impliquent que les deux premiers numéros de la version sont identiques.

* Installez toujours le dernier module complémentaire Service Pack/Feature Pack disponible. Il est vivement recommandé car il comprend des correctifs et améliorations concernant la sécurité, les performances, la stabilité et les clients clés publiés depuis la disponibilité générale de la version de base.
* Chaque fois que vous installez un Service Pack ou effectuez une mise à niveau d&#39;une version précédente d&#39;AEM vers une version plus récente, installez toujours le module complémentaire AEM [!DNL Forms] correspondant. Par exemple, si vous effectuez la mise à niveau d’AEM package de module complémentaire 6.1 [!DNL Forms] version 2.2.4 vers AEM 6.1 Service Pack 1, après avoir installé l’ 6.1 Service Pack 1, installez le package de module complémentaire [!DNL Forms] version 2.4.50.
* La désinstallation du module complémentaire [!DNL Forms] AEM n&#39;est pas prise en charge.
